# Requirements and Traceability

Canonical local source: `requirements/catalog.json`, containing FR-01–FR-104 and NFR-01–NFR-18. The original SRS and workflow source files cited by older documentation are absent; content beyond the catalog is **INSUFFICIENT DATA TO VERIFY**. Statuses reflect the implementation and executed evidence currently present in this repository. No requirement is marked verified solely because a UI route exists.

## Status definitions

- **Implemented and verified:** complete implementation with passing automated evidence.
- **Implemented but not verified:** implementation exists without sufficient executed evidence.
- **Partially implemented:** at least one required layer or branch remains.
- **Blocked:** external decision, provider, credential, policy, or content prevents completion.
- **Not implemented:** no supporting implementation exists.

## Approved scope decisions

- Cash is the only enabled payment method; non-cash methods are visible but unavailable and cannot create transactions.
- Worker verification fees and advertising are excluded. Recommendation priority is administrator-controlled and cannot override suitability.
- Each Auth identity has one immutable primary role. Historical secondary profiles remain stored but cannot grant access, and session role switching is unavailable.
- General Trash permanent deletion uses an entity allowlist, AAL2 and exact typed confirmation. Retained business records remain protected. User and Worker account deletion additionally requires exact email confirmation; Administrator deletion remains prohibited.
- Provider test adapters are not production substitutes. Production fails closed when required provider bindings are absent.
